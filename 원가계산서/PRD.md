## 0. 한눈에 보기

### 제품 한 문장 정의

사용자가 **원가계산서 Excel 파일**과 **제비율·노임단가 등 계산기준 Excel 파일**을 업로드하면, 시스템이 원가 항목·수식·요율·합계 등을 자동 검증하고 **검증결과와 판단근거를 같은 웹 화면에서 항목별로 보여주는 서비스**입니다. 표준품셈·지침 등 보조 근거자료는 필요 시 PDF로 추가할 수 있습니다.

### 입력

1. 필수 입력: 원가계산서 `.xlsx`
2. 필수 입력: 제비율표, 노임단가표 등 계산기준 `.xlsx`
3. 선택 입력: 조달청 표준품셈, 가격 산정지침 등 보조 기준자료 `.pdf`
4. 사용자 선택정보: 발주유형, 기준일, 공사기간 등 적용조건

### 출력

- 웹 검증 결과화면
- 정상·주의·오류·확인 필요 건수
- 오류 위치와 입력값
- 시스템 기대값과 차액
- 적용한 계산식
- 판단근거가 된 기준자료 파일명·시트·셀 또는 PDF 문서명·페이지
- 기준자료 원본값 또는 근거 원문 발췌
- 자동판정 신뢰도 및 사람이 확인해야 할 사항
- 후속 공고문 생성 기능에서 재사용할 수 있는 `validatedProject` JSON

### 핵심 변경사항

- 공고문 생성 기능은 삭제가 아니라 이번 MVP 범위에서 제외하고 Phase 2 확장으로 분리
- RFP 입력 기능은 이번 MVP 범위에서 제외
- 검증결과 Excel 추출 기능 삭제
- 공사·용역·물품 공통 적용 구조로 확장
- 공사 원가계산서 검증을 우선 구현
- 제비율·노임단가 기준자료 입력 형식을 Excel로 확정
- 표준품셈·지침 등 보조 근거자료는 PDF 입력을 선택 지원
- 검증결과와 판단근거를 웹 화면에서 나란히 표시
- 개발기간을 3일에서 2일 시연형으로 축소

# 1. 제품 배경

## 1.1 해결하려는 문제

원가계산서는 하나의 표처럼 보이지만 실제로는 다음 요소가 복합적으로 연결된 계산 문서입니다.

- 여러 시트 간 셀 참조
- 병합셀과 다단 헤더
- 수량 × 단가, 소계, 제경비, 일반관리비, 이윤, 부가가치세 등 단계별 계산
- 공사종류·금액·기간에 따라 달라지는 제비율
- 원 단위·백 원 단위·천 원 단위 절사 및 반올림
- 노임단가·가격조사·일위대가·표준품셈 등 하위 산출근거
- 견적가와 설계가 비교 후 최종금액 선택
- 적용 제외, 하한금액, 실적정산 등 예외 조건

현재는 검토자가 원가계산서 Excel, 제비율 Excel, 노임단가 Excel, 필요 시 표준품셈 PDF를 동시에 열어 직접 비교합니다. 이 과정에서 다음 문제가 발생합니다.

1. 수식 참조나 합계 오류를 놓칠 수 있습니다.
2. 요율 숫자는 맞지만 산출기초가 틀린 경우를 찾기 어렵습니다.
3. 최신 기준 또는 적절한 금액구간을 적용했는지 확인하기 어렵습니다.
4. 오류를 발견한 뒤에도 판단근거를 다시 수작업으로 작성해야 합니다.
5. 파일 양식이 달라질 때마다 검토 방식이 달라집니다.
6. 최종금액에서 하위 산출근거까지 역추적하는 데 시간이 많이 듭니다.

## 1.2 제품 비전

> 원가계산서와 계산기준자료를 업로드하면, 시스템이 숫자와 계산 구조를 재검증하고 **어디가 맞거나 틀렸는지, 왜 그렇게 판단했는지**를 근거와 함께 보여줍니다.

이번 MVP는 향후 공고문 자동생성 기능의 선행 단계입니다. 즉, 먼저 원가계산서의 금액·요율·산출기초·근거를 신뢰 가능한 구조로 검증하고, 그 결과를 `validatedProject`로 보존합니다. Phase 2에서는 이 `validatedProject`와 공고조건 입력값을 받아 공고문 초안, 법령·필수조항 추천, 미리보기·출력 기능으로 확장합니다.

## 1.3 핵심 제품 원칙

### 원칙 1. 오류 여부와 근거를 분리하지 않습니다.

모든 오류·주의·확인 필요 결과에는 반드시 다음 중 하나 이상의 근거가 연결되어야 합니다.

- Excel 원본 수식
- 시스템 재계산식
- 동일 시트 또는 다른 시트의 참조 셀
- 기준자료 Excel의 시트·셀 또는 선택 PDF의 페이지·표·문구
- 사용자 입력 적용조건

### 원칙 2. 엑셀을 단순 텍스트로만 바꾸지 않습니다.

Excel을 바로 Markdown으로 변환하면 셀 주소, 수식, 병합범위, 숨김행, 시트 간 참조가 손실될 수 있습니다. 검증에는 다음 정보를 보존하는 표준 중간 데이터가 필요합니다.

- 시트명
- 셀 주소
- 표시값
- 원본 수식
- Excel에 저장된 마지막 계산값
- 병합범위
- 숫자 표시형식
- 행·열 숨김 여부
- 다른 시트 참조관계

Markdown은 LLM이 내용을 이해하거나 사용자에게 설명하기 위한 보조 데이터로만 사용합니다.

### 원칙 3. 숫자 확정은 규칙 엔진이 담당합니다.

LLM은 항목명 분류, 표준품셈 PDF 문장 해석, 근거 설명을 보조할 수 있으나 다음 판단은 Python 규칙 엔진이 수행합니다.

- 사칙연산
- 합계 및 소계
- 비율 적용
- 절사·반올림
- 금액구간 판정
- 날짜 및 기간 조건
- 차액 계산

AI API를 붙이더라도 역할은 검증 엔진의 보조 레이어로 제한합니다. AI는 항목명 매핑 후보, 기준자료 적합성 판단, 검증불가 사유 설명, 선택 PDF 근거 해석, 사용자용 판단근거 문장 생성을 돕습니다. 최종 금액, 요율 적용, 오류 여부 확정은 재현 가능한 규칙 엔진과 기준자료 셀 근거를 기준으로 결정합니다.

### 원칙 4. 자동판정이 어려우면 억지로 확정하지 않습니다.

근거가 부족하거나 기준자료 추출 결과가 불명확한 경우 상태를 `확인 필요`로 표시하고, 무엇을 확인해야 하는지 안내합니다.

# 2. 제품 목표와 범위

## 2.1 제품 목표

### G-01. 원가계산서 구조 인식

공사·용역·물품 원가계산서에서 시트, 표 영역, 항목명, 수량, 단가, 금액, 요율, 수식 및 비고를 식별합니다.

### G-02. 공통 산술 검증

발주유형과 관계없이 다음을 검증합니다.

- 수량 × 단가 = 금액
- 세부 항목 합계 = 소계
- 공급가액 + 부가가치세 = 총액
- 시트 간 금액 일치
- 수식 누락 또는 값 하드코딩
- 참조 오류와 비정상 범위
- 끝수처리 일관성

### G-03. 공사 원가 심화검증

공사 원가계산서에 대해서는 다음 항목을 우선 검증합니다.

- 직접재료비·간접재료비
- 직접노무비·간접노무비
- 산재보험료·고용보험료 등 제보험료
- 산업안전보건관리비
- 기타경비
- 일반관리비
- 이윤
- 근로자재해보장책임보험 등 선택 항목
- 표준품셈·일위대가·노임단가 연계
- 공사기간 또는 금액구간에 따른 적용·제외 조건

### G-04. 기준자료 구조화

업로드한 제비율 Excel과 노임단가 Excel에서 기준일, 적용대상, 금액구간, 요율, 산출기초, 직종명, 단가 및 예외조건을 추출해 기준정보로 만듭니다. 표준품셈 PDF가 입력된 경우에는 문서명, 페이지, 품셈 항목, 공량 및 원문 위치를 보조 기준정보로 구조화합니다.

### G-05. 판단근거 제시

모든 검증결과에 실제값, 기대값, 차액, 적용식, 원가계산서 Excel 위치, 기준자료 Excel 위치 또는 선택 PDF 근거 위치를 표시합니다.

### G-06. 웹 중심 결과 확인

검증결과를 별도 파일이 아닌 웹 화면에서 조회하고, 필터·정렬·상세근거 열람이 가능하도록 합니다.

## 2.2 발주유형별 적용 범위

| 구분 | 제품 전체 방향 | 2일 시연형 구현 깊이 |
|---|---|---|
| 공사 | 공통 산술 + 제비율 Excel + 노임단가 Excel + 선택 표준품셈 PDF + 일위대가 + 기간·금액 조건 | 핵심 시연 대상. 공통 산술, 주요 제비율, 주요 노임단가 검증 구현 |
| 용역 | 공통 산술 + 인건비·경비·일반관리비·이윤 등 유형별 기준 | 공통 산술·합계·부가세 검증, 유형 분류 구조 구현 |
| 물품 | 수량·단가·금액·견적 비교·부가세·총액 검증 | 공통 산술·합계·부가세 검증 구현 |

제품 구조는 세 유형을 모두 수용하되, 2일 내 시연 완성도를 확보하기 위해 공사 검증을 가장 깊게 구현합니다.

# 3. 대상 사용자

## 3.1 주요 사용자

### 원가계산서 작성자

작성한 파일의 수식, 합계, 요율 적용 오류를 제출 전에 확인합니다.

### 계약·구매·발주 담당자

제출받은 공사·용역·물품 원가계산서의 주요 오류와 확인 필요 항목을 빠르게 선별합니다.

### 기술·원가 검토자

최종금액에서 하위 산출근거까지 계산 경로를 추적하고, 적용한 기준자료가 적절한지 확인합니다.

### 기준정보 관리자

제비율·노임단가 Excel과 선택 입력된 표준품셈 PDF가 올바르게 구조화되었는지 검수합니다.

## 3.2 사용자 핵심 요구

- 어느 시트의 어느 항목이 문제인지 즉시 확인하고 싶습니다.
- 틀렸다는 말뿐 아니라 맞는 계산값을 보고 싶습니다.
- 어떤 기준자료 파일의 어느 시트·셀 또는 PDF 페이지를 근거로 판단했는지 알고 싶습니다.
- 계산이 맞아도 잘못된 요율이나 산출기초를 적용한 경우를 찾고 싶습니다.
- 시스템이 확신하지 못하는 부분은 명확히 구분해서 보고 싶습니다.

## 4.1 간단형 원가계산서

|---|---:|
| 시트 수 | 4개 |
| 시트명 | 표지, 원가계산서, 견적서, 제비율 적용기준 |
| 수식 셀 수 | 62개 |
| 병합 범위 수 | 63개 |

주요 특징은 다음과 같습니다.

- 원가계산서에서 재료비·노무비·제경비·일반관리비·이윤 등을 계산합니다.
- 적용요율을 별도 `제비율 적용기준` 시트에서 참조합니다.
- 표지의 추정가격·부가가치세·합계가 원가계산서의 결과를 참조합니다.
- `INT` 등을 이용한 절사와 별도 금액 조정이 포함됩니다.

## 4.2 복잡형 원가계산서

| 구분 | 분석 결과 |
|---|---:|
| 시트 수 | 18개 |
| 수식 셀 수 | 931개 |
| 병합 범위 수 | 391개 |
| 핵심 시트 | 견적가 및 설계 비교, 추정가격내역서, 공사내역서, 수량, 일위대가, 단가, 가격조사, 노임단가, 제비율 적용기준 |

주요 계산 흐름은 다음과 같습니다.

```text
노임단가 / 가격조사 / 단가 / 수량
            ↓
         일위대가
            ↓
공사내역서 + 간접재료비 + 별도 산출내역
            ↓
       설계 추정가격
            ↓
  견적 추정가격과 비교
            ↓
       최종 추정가격
            ↓
       부가가치세·합계
```

복잡형은 최종금액만 확인해서는 오류 여부를 판단하기 어렵습니다. 시트 간 계산 계보와 각 단계의 끝수처리 방식을 함께 보존해야 합니다.

## 4.3 샘플 분석이 제품설계에 주는 시사점

1. 첫 번째 시트만 읽는 방식은 사용할 수 없습니다.
2. 표시된 값과 원본 수식을 동시에 읽어야 합니다.
3. 병합셀을 해제하거나 단순 행 데이터로 바꾸기 전에 원본 위치를 보존해야 합니다.
4. 동일 항목도 견적 방식과 설계 방식에 따라 계산식이 다를 수 있습니다.
5. 제비율 숫자뿐 아니라 산출기초와 적용조건을 함께 검증해야 합니다.
6. 결과화면에서 상위 금액과 하위 근거를 연결해 보여줘야 합니다.

- 기준자료명 및 기준일
- 분석한 시트 수
- 표준화된 원가계산서 행 수

- 주의 건수

핵심 확인사항은 단순히 오류를 나열하지 않고, 사용자가 먼저 조치해야 하는 항목을 우선순위로 보여줍니다.

## 6.4 정규화 원가계산서 리포트

검증결과 아래에는 원본 Excel 양식과 관계없이 동일한 컬럼으로 정규화된 원가계산서 표를 표시합니다. 이 표는 JSON 원문 미리보기가 아니라 검토자가 Excel처럼 훑어볼 수 있는 표 형태여야 합니다.

기본 열은 다음과 같습니다.

| 열 | 내용 |
|---|---|
| 상태 | 정상 / 주의 / 오류 / 확인 필요 / 검증 불가 |
| 원본구간 | 원본 표의 계층상 구간. 재료비, 노무비, 경비, 일반관리비, 이윤, 별도비목, 집계 등 |
| 비용성격 | 실제 검증 성격. 제비율성 경비, 노임성 직접비, 별도 직접비, 실적정산성 항목 등 |
| 검증정책 | RATE_CHECK, LABOR_RATE_CHECK, INTERNAL_CONSISTENCY, REFERENCE_REQUIRED, MANUAL_REVIEW 등 |
| 표준항목 | 표준 코드 또는 표준 항목명 |
| 원본항목 | Excel에 기재된 원래 항목명 |
| 산출기초명 | 원본 행 또는 수식에서 식별한 산출기초명 |
| 산출기초값 | 같은 행, 같은 시트, 다른 시트에서 추적한 기준 금액 |
| 요율 | 원본 요율 값 |
| 요율출처 | 요율이 입력된 셀 또는 기준자료 위치 |
| 금액 | Excel에 저장된 원본 금액 |
| 계산값 | 시스템이 재계산한 기대 금액 |
| 차이 | 원본 금액과 계산값의 차이 |
| 계산셀 | 실제 계산이 수행된 셀. 표시 금액 셀과 다를 수 있음 |
| 해결상태 | SAME_ROW, FORMULA_TRACE, CROSS_SHEET_TRACE, UNRESOLVED 등 |
| 수식상수 | 수식 안에 숨어 있는 보정값, 최소금액, 절사 단위 등 |
| 추적경로 | 참조 셀과 시트 간 계산 흐름 |
| 비고 | 원본 비고 또는 시스템 판단 메모 |

정규화 시 `원본구간`과 `비용성격`을 분리합니다. 원본 표에서 A열 등 최상위 위치에 새 항목이 시작되면 직전 구간을 자동 상속하지 않고 `별도비목` 또는 독립 항목으로 분류합니다. 예를 들어 `비파괴검사`가 원본에서 경비 블록 안에 있지 않다면, 항목명이 검사성 비용처럼 보여도 `경비`로 밀어 넣지 않습니다.

섹션 경계는 원본 표의 소계 행을 기준으로 판단합니다. `재료비 소계` 다음부터 `노무비 소계`까지는 `노무비`, `노무비 소계` 다음부터 `경비 소계`까지는 `경비`로 분류합니다. `경비 소계`가 나온 뒤에는 경비 블록을 종료하며, 그 아래의 `일반관리비`, `이윤`, 공사손해보험료, 폐기물처리비, 비파괴검사 등 후속 항목은 원본 위치와 근거에 따라 `일반관리비`, `이윤`, `별도비목`, `집계` 등으로 분류하고 `경비`를 상속하지 않습니다.

필터 조건은 다음과 같습니다.

- 상태별
- 시트별
- 검증유형별
- 원가항목별
- 원본구간별
- 비용성격별
- 검증정책별
- 해결상태별
- 참조 시트별
- 공사·용역·물품 공통검증 / 유형별 심화검증

## 6.5 검증근거 상세 패널

사용자가 결과행을 클릭하면 오른쪽 패널 또는 하단 확장영역에 다음을 표시합니다.

### 1. 판단 요약

> 입력된 산재보험료 요율 3.20%는 선택한 기준자료의 적용요율 3.56%와 일치하지 않습니다.

### 2. Excel 원본정보

- 파일명
- 시트명
- 셀 주소
- 입력값
- 원본 수식
- 참조한 셀
- 계산셀
- 산출기초 추적 경로
- 요율 출처
- 수식 내 상수

### 3. 시스템 계산과정

```text
산출기초: 직접노무비 102,500,000원
기준요율: 3.56%
계산값: 102,500,000 × 0.0356 = 3,649,000원
적용 끝수처리: 원 단위 절사
기대값: 3,649,000원
입력값: 3,280,000원
차액: -369,000원
```

### 4. 기준자료 판단근거

- 기준자료 파일명
- Excel 시트명 및 셀 주소
- PDF 문서명
- 기준일 또는 버전
- 페이지 번호
- 표 제목 또는 조항명
- 원문 발췌
- 추출된 적용조건

### 5. 판정 신뢰도

- 높음: 숫자·조건·근거 위치가 명확히 일치
- 보통: 항목은 일치하나 조건 일부가 사용자 입력에 의존
- 낮음: 기준자료 구조 또는 항목 매핑이 불명확

### 6. 권장 조치

- 요율 수정
- 산출기초 확인
- 공사기간 확인
- 적용 제외 사유 확인
- 기준자료 버전 확인

## 6.6 원본 추적 및 보조 정보

가능한 경우 결과화면에 간단한 Excel 원본 미리보기와 참조 시트 정보를 함께 제공하고, 결과행을 클릭하면 해당 셀 또는 행을 강조합니다.

2일 시연형에서는 원본 Excel 전체를 스프레드시트처럼 재현하지 않고 다음 정도로 제한합니다.

- 해당 시트명 표시
- 문제 셀을 포함한 앞뒤 2~3개 행 미리보기
- 문제 셀 강조
- 수식·값 표시 전환
- 참조된 시트 목록
- 원가계산서 정규화 행 수
- 미매핑 항목 수
- 같은 행 / 같은 시트 / 다른 시트 추적 건수

## 6.7 빈 결과와 오류 상태

### 모든 검증 통과

> 자동검증 범위에서 확인된 오류가 없습니다. 다만 시스템이 지원하지 않는 항목은 별도 확인이 필요합니다.

### 기준자료 근거 추출 실패

> 기준자료에서 해당 항목의 요율 또는 적용조건을 안정적으로 추출하지 못했습니다. 숫자 오류로 확정하지 않고 ‘확인 필요’로 분류했습니다.

### Excel 수식값 없음

> 수식은 존재하지만 Excel에 저장된 최근 계산값이 없습니다. 시스템 재계산이 가능한 항목만 검증했습니다.

# 7. 검증 규칙

## 7.1 상태 정의

| 상태 | 정의 |
|---|---|
| 정상 | 입력값과 기대값이 허용오차 내에서 일치하고 근거가 충분함 |
| 주의 | 계산은 일치하지만 수식 하드코딩, 미사용 참조, 기준일 불명확 등 검토 권고사항이 있음 |
| 오류 | 산술·수식·요율·산출기초·적용조건이 명확히 기준과 불일치함 |
| 확인 필요 | 근거 또는 적용조건이 부족하여 자동 확정할 수 없음 |

## 7.2 공통 산술 규칙

### V-COM-001 수량 × 단가 검증

```text
기대금액 = 수량 × 단가
```

검증 시 표시형식에 따른 소수점, 반올림 또는 절사 여부를 함께 고려합니다.

### V-COM-002 소계 검증

세부항목 합계와 소계 셀을 비교합니다.

### V-COM-003 총액 검증

공급가액, 부가가치세, 총액의 관계를 검증합니다.

### V-COM-004 시트 간 금액 일치

표지·요약표·상세내역서 등에서 동일 의미로 사용된 금액이 일치하는지 확인합니다.

### V-COM-005 수식 누락 및 값 하드코딩

주변 행이 수식인데 특정 셀만 숫자값으로 직접 입력된 경우 주의 또는 오류로 표시합니다.

### V-COM-006 참조 오류

`#REF!`, 순환참조 의심, 존재하지 않는 시트 참조, 비정상 범위 등을 확인합니다.

### V-COM-007 끝수처리 검증

`ROUND`, `ROUNDDOWN`, `INT` 등 원본 수식과 기준 규칙의 끝수처리가 일치하는지 확인합니다.

## 7.3 공사 심화 규칙

### V-CON-001 요율 값 검증

입력요율과 제비율 Excel에서 구조화한 기준요율을 비교합니다.

### V-CON-002 요율 산출기초 검증

요율이 맞더라도 직접노무비, 노무비, 재료비+노무비 등 산출기초가 기준과 다른 경우 오류로 표시합니다.

### V-CON-003 금액구간 검증

공사금액에 해당하는 올바른 구간의 요율을 적용했는지 확인합니다.

### V-CON-004 공사기간 조건 검증

30일 미만 제외 등 기간 조건에 따라 해당 항목의 적용 또는 제외가 올바른지 확인합니다.

### V-CON-005 일반관리비·이윤 검증

기준요율, 적용대상 금액, 상한 또는 적용방식을 확인합니다.

### V-CON-006 표준품셈 연계 검증

표준품셈 PDF가 입력된 경우 일위대가의 품명·규격·단위·공량이 표준품셈 PDF와 매핑되는지 확인합니다. 2일 시연형에서는 사전에 선정한 일부 대표항목만 지원하며, PDF가 없으면 해당 검증은 `미수행`으로 표시합니다.

### V-CON-007 노임단가 연계 검증

직종명과 단가를 노임단가 Excel 기준과 비교합니다. 직종명 매핑이 불명확하면 확인 필요로 표시합니다.

## 7.4 용역·물품 규칙

2일 시연형에서는 다음 공통 수준을 지원합니다.

### 용역

- 인원 × 투입기간 × 단가
- 직접인건비·경비·일반관리비·이윤 합계
- 부가가치세와 총액
- 동일 항목의 시트 간 금액 일치

### 물품

- 수량 × 단가
- 품목별 금액 합계
- 공급가액·부가가치세·총액
- 견적 또는 가격조사 값과 선택단가의 일관성

유형별 세부 요율과 법정 기준은 후속 단계에서 규칙팩으로 확장합니다.

## 7.5 허용오차

모든 금액을 부동소수점 `float`로 계산하지 않고 `Decimal`을 사용합니다.

기본 허용오차는 다음과 같이 설정합니다.

- 원 단위 계산: 0원
- 표시값 반올림이 있는 경우: 표시단위 이내
- 요율: 기준표의 소수 자릿수에 맞춰 비교
- 시스템이 끝수처리를 판단하지 못한 경우: 확인 필요

# 8. 기준자료 처리

## 8.1 기준자료 입력 구분

2일 시연형에서는 기준자료를 다음과 같이 구분합니다.

|---|---|---|---|

# 10. 파싱 라이브러리 권고안

## 10.1 최종 권고 조합

### Excel 주 파서: `openpyxl`

용도:

- `.xlsx` 워크북과 시트 읽기
- 셀 주소·수식·저장값 추출
- 병합셀·표시형식·숨김정보 확인
- 시트 간 참조관계 분석

사용 방식:

```text
1회차: data_only=False → 원본 수식 읽기
2회차: data_only=True  → Excel에 저장된 마지막 계산값 읽기
```

주의사항:

- openpyxl은 Excel 계산엔진이 아니므로 수식을 새로 계산하지 않습니다.
- 따라서 주요 원가 공식은 규칙 엔진에서 직접 재계산해야 합니다.
- 수식과 저장값을 모두 보존하려면 동일 파일을 두 방식으로 읽습니다.

### Excel 보완 파서: Python `zipfile` + XML

`.xlsx`는 OOXML ZIP 패키지이므로 다음 항목은 원본 XML을 직접 확인하는 보완 로직을 둡니다.

- 외부 링크
- 정의된 이름
- 수식 캐시
- 계산모드
- openpyxl에서 누락될 수 있는 일부 메타데이터

2일 시연형에서는 외부참조 여부와 정의된 이름 오류 정도만 점검합니다.

### 선택 PDF 주 파서: `PyMuPDF`

용도:

- 페이지별 텍스트 추출
- 단어·블록 좌표 추출
- 표 탐지
- 페이지·근거 위치 보존
- 근거 원문 하이라이트용 좌표 확보

표준품셈 PDF의 “몇 페이지 어느 영역”인지 결과화면에 보여줘야 하므로, 단순 텍스트만 반환하는 도구보다 좌표정보를 제공하는 파서가 적합합니다.

### PDF 표 추출 보완: `pdfplumber`

용도:

- 선 기반 표 추출 보완
- 문자·선·사각형 단위 분석
- 표 탐지 설정 조정
- 시각적 디버깅

PyMuPDF에서 표준품셈 PDF의 표 구조가 안정적으로 나오지 않는 경우에 한해 fallback으로 사용합니다.

### LLM용 보조 변환: `MarkItDown`

용도:

- PDF 또는 Excel 전체 내용을 LLM이 읽기 쉬운 Markdown으로 변환
- 문서 요약과 항목명 후보 생성
- 규칙 설명문 생성

비권장 용도:

- Excel 숫자 검증의 원본 데이터
- 셀 주소·수식·시트관계의 기준 데이터
- PDF 표의 정확한 행·열 판정

즉, MarkItDown은 **보조 텍스트 변환기**로 사용하고, 실제 검증 데이터는 원가계산서 Excel, 제비율 Excel, 노임단가 Excel, 선택 표준품셈 PDF에서 각각 생성한 구조화 결과를 사용합니다.

## 10.2 2일 시연형 기술선택

| 영역 | 권고 기술 | 선정 이유 |
|---|---|---|
| 백엔드 | Python + FastAPI | Excel·PDF 라이브러리 사용이 쉽고 기존 HTML과 연결 가능 |
| 프론트엔드 | 기존 HTML/CSS/JavaScript 수정 | 새 프레임워크 학습·빌드 없이 결과화면 구현 가능 |
| Excel | openpyxl | 수식·셀 주소·병합·표시형식 보존 |
| 선택 PDF | PyMuPDF | 표준품셈 PDF의 텍스트·표·좌표 추출 |
| 선택 PDF fallback | pdfplumber | 표준품셈 PDF의 복잡한 표 디버깅과 보완 |
| 데이터 모델 | Pydantic | 입력·출력 JSON 구조 검증 |
| 금액 계산 | Python Decimal | 금액·요율 계산 오차 방지 |
| 임시 상태 | 메모리 또는 세션 JSON | 2일 시연에서 DB 불필요 |
| LLM 보조 | 선택적 API 연동 | 항목명 매핑·근거 설명에만 제한적으로 사용 |

## 10.3 대안 검토

### SheetJS

브라우저에서 Excel 미리보기를 만드는 데는 유용하지만, 이번 프로젝트는 원가계산서·기준자료 Excel 파싱과 선택 PDF 파싱을 함께 처리해야 하므로 주 파서는 Python으로 통일하는 편이 빠릅니다. 브라우저 미리보기가 꼭 필요할 때만 보조적으로 사용합니다.

### pandas

정형 표 분석에는 유용하지만 병합셀, 수식, 서식, 셀 주소, 시트 간 참조 보존이 핵심인 이번 프로젝트의 주 파서로는 부족합니다. 정규화 이후 집계용으로만 사용합니다.

### Camelot 또는 Tabula

표 형태가 일정한 PDF에서는 유용하지만 추가 의존성과 설정 부담이 있습니다. 2일 시연에서는 PyMuPDF를 우선 사용하고, 표준품셈 PDF에서 추출률이 낮을 때만 도입합니다.

### Docling·Unstructured 등 문서 파이프라인

장기적으로는 다양한 PDF 레이아웃과 OCR까지 확장할 때 검토할 수 있습니다. 2일 시연에서는 설정과 모델 의존성이 커서 우선순위에서 제외합니다.

# 12. 기능 요구사항

## 12.1 파일 입력

### FR-001 Excel 업로드

- `.xlsx` 파일을 업로드할 수 있어야 합니다.
- 암호화 파일, 손상 파일, 빈 파일을 구분해야 합니다.
- 파일명, 시트 수, 수식 수를 표시해야 합니다.

### FR-002 발주유형 선택

- 공사·용역·물품 중 하나를 선택해야 합니다.
- 시스템 자동추정값을 보여줄 수 있으나 사용자가 최종 선택합니다.

### FR-003 기준자료 업로드

- 제비율 Excel과 노임단가 Excel을 필수 기준자료로 업로드할 수 있어야 합니다.
- 표준품셈 PDF는 선택 입력으로 업로드할 수 있어야 합니다.
- 기준자료 유형, 파일명, 시트 수, 기준일, 추출 항목 수를 표시해야 합니다.
- 선택 PDF는 페이지 수와 텍스트 추출 가능 여부를 추가로 표시해야 합니다.

### FR-004 적용조건 입력

- 기준일, 금액, 기간, 공사종류 등 검증에 필요한 조건을 입력할 수 있어야 합니다.
- Excel에서 추출한 값은 자동 채우고 사용자가 확인합니다.

## 12.2 파싱과 매핑

### FR-010 Excel 구조 파싱

- 모든 시트의 사용영역, 셀 주소, 값, 수식, 표시형식, 병합범위를 읽어야 합니다.

### FR-011 시트 역할 분류

시트를 다음과 같은 역할로 분류할 수 있어야 합니다.

- 표지·요약
- 원가계산서
- 공사내역서
- 수량
- 일위대가
- 단가·가격조사
- 노임단가
- 제비율 기준
- 기타

### FR-012 항목명 표준화

띄어쓰기와 유사표현을 표준 원가항목으로 매핑해야 합니다.

### FR-013 기준자료 추출

제비율 Excel의 요율·산출기초·적용조건·예외조건·출처 위치와 노임단가 Excel의 직종명·단가·기준일·출처 위치를 구조화해야 합니다. 표준품셈 PDF가 입력된 경우 품셈 항목·공량·적용조건·페이지 위치를 선택적으로 구조화해야 합니다.

## 12.3 검증

### FR-020 공통 산술검증

공사·용역·물품에 대해 수량×단가, 소계, 총계, 부가세를 검증해야 합니다.

### FR-021 수식·참조검증

수식 누락, 하드코딩, 참조 오류, 시트 간 불일치를 탐지해야 합니다.

### FR-022 공사 요율검증

선정된 제비율 Excel 기준자료와 원가계산서 Excel의 요율을 비교해야 합니다.

### FR-023 산출기초검증

요율이 적용된 기준금액이 제비율 Excel의 산출기초 정의와 일치하는지 확인해야 합니다.

### FR-024 적용조건검증

금액·기간·공사종류 조건에 맞는 규칙을 선택해야 합니다.

### FR-025 끝수처리검증

절사·반올림 방식이 원가계산서와 기준 규칙에 맞는지 확인해야 합니다.

## 12.4 결과화면

### FR-030 검증요약 표시

로딩 완료 후 결과화면 최상단에 종합 검증상태, 정상·주의·오류·확인 필요·검증 불가 건수, 핵심 확인사항 3~5개를 표시해야 합니다.

### FR-031 결과 필터링

상태, 시트, 검증유형, 원가항목, 해결상태, 참조 시트로 필터링할 수 있어야 합니다.

### FR-032 결과행 표시

입력값, 기대값, 차액, Excel 위치, 판단요약을 표시해야 합니다.

### FR-033 근거 상세 표시

결과행 선택 시 계산과정, 원본 수식, 계산셀, 참조 셀, 산출기초 추적 경로, 요율 출처, 노임단가 출처, 수식 내 상수, 기준자료 Excel 시트·셀, 선택 PDF 페이지·원문 발췌를 표시해야 합니다.

### FR-034 정규화 원가계산서 리포트 표시

원본 Excel 양식과 관계없이 표준 컬럼으로 정규화된 원가계산서 표를 결과화면에 표시해야 합니다. 표에는 원본구간, 비용성격, 검증정책, 표준항목, 원본항목, 산출기초값, 요율, 요율출처, 금액, 계산값, 차액, 계산셀, 해결상태, 추적경로가 포함되어야 합니다.

### FR-035 원본 주변행 표시

문제 셀 주변의 Excel 행을 간단한 표로 보여줘야 합니다.

### FR-036 신뢰도 표시

근거 추출과 항목 매핑의 신뢰도를 보여줘야 합니다.

### FR-037 수식 추적 상태 표시

각 정규화 행은 산출기초 해결상태를 표시해야 합니다. 최소 상태값은 `SAME_ROW`, `FORMULA_TRACE`, `CROSS_SHEET_TRACE`, `CROSS_SHEET_UNRESOLVED`, `FORMULA_UNRESOLVED`, `UNRESOLVED`를 포함합니다.

# 15. 시연 시나리오

## 15.1 시나리오 A — 정상 또는 간단형 파일

1. 간단형 Excel 업로드
2. 발주유형 `공사` 선택
3. 시연용 제비율 Excel과 노임단가 Excel 업로드
4. 선택 표준품셈 PDF 업로드
5. 기준일·공사기간 입력
6. 검증 실행
7. 합계, 주요 요율, 주요 노임단가 검증결과 확인
8. 결과행을 클릭해 원본 수식과 기준자료 셀 근거 확인

## 15.2 시나리오 B — 오류 삽입 파일

시연용 복사본에 다음 오류를 미리 삽입합니다.

- 수량 × 단가 금액 1건 불일치
- 제비율 1건 오적용
- 산출기초 1건 잘못 참조
- 부가가치세 또는 최종합계 1건 불일치

시연 흐름:

```text
오류 파일 업로드
→ 오류 4건 표시
→ 요율 오류 선택
→ 입력요율·기준요율·차액 확인
→ 제비율 Excel의 기준 셀 근거 확인
→ Excel 셀과 원본 수식 확인
```

## 15.3 시나리오 C — 용역 또는 물품

간단한 용역 또는 물품 샘플을 업로드해 다음만 확인합니다.

- 발주유형 선택 가능
- 수량·단가·금액 검증
- 소계·부가세·총액 검증
- 결과화면과 근거 패널이 동일 구조로 작동

# 16. 수용 기준

## 16.1 필수 수용 기준

- [ ] 공사·용역·물품 발주유형을 선택할 수 있습니다.
- [ ] 제공된 간단형·복잡형 Excel을 업로드할 수 있습니다.
- [ ] 모든 시트명과 셀 수식·저장값을 읽을 수 있습니다.
- [ ] 제비율 Excel과 노임단가 Excel을 기준자료로 업로드할 수 있습니다.
- [ ] 기준자료 Excel에서 기준일, 항목명, 요율, 산출기초, 직종명, 단가, 셀 위치를 추출합니다.
- [ ] 선택 표준품셈 PDF를 업로드할 수 있습니다.
- [ ] 수량×단가, 소계, 부가세, 총액 중 3종 이상을 검증합니다.
- [ ] 공사 제비율 3종 이상을 기준 Excel과 비교합니다.
- [ ] 노임단가 3종 이상을 기준 Excel과 비교합니다.
- [ ] 오류 결과에 시트명·셀 주소가 표시됩니다.
- [ ] 오류 결과에 입력값·기대값·차액이 표시됩니다.
- [ ] 오류 결과 옆에 판단근거가 표시됩니다.
- [ ] 판단근거에 기준자료 파일명·시트명·셀 주소가 표시됩니다.
- [ ] 자동판정 불가 항목을 확인 필요로 구분합니다.
- [ ] 결과는 웹 화면에서만 확인하며 Excel 추출 기능은 제공하지 않습니다.

## 16.2 시연 성공 기준

- 사용자가 업로드 후 3분 이내에 결과를 이해할 수 있습니다.
- 오류행을 클릭했을 때 왜 오류인지 추가 설명 없이 파악할 수 있습니다.
- 검토자가 기준자료 Excel의 셀 근거로 판정을 재확인할 수 있습니다.
- 정상·주의·오류·확인 필요 상태가 시각적으로 명확히 구분됩니다.

# 17. 비기능 요구사항

## 17.1 성능

- 20MB 이하 Excel과 30MB 이하 선택 PDF를 시연 대상으로 합니다.
- 샘플 파일 기준 업로드부터 결과 표시까지 목표 30초 이내입니다.
- 진행단계 문구를 표시합니다.

```text
Excel 구조 분석 중
기준자료 Excel 추출 중
선택 PDF 추출 중
원가 항목 매핑 중
검증 규칙 실행 중
판단근거 생성 중
```

## 17.2 정확성

- 금액 계산에는 `Decimal`을 사용합니다.
- 원본값을 수정하지 않습니다.
- 모든 자동판정은 적용한 규칙 ID와 근거를 보존합니다.
- 기준자료 추출 신뢰도가 낮으면 오류로 확정하지 않습니다.

## 17.3 보안

2일 시연형에서는 파일을 서버 디스크에 장기 보관하지 않습니다.

- 세션 종료 시 임시파일 삭제
- 파일명 외 개인정보 로그 최소화
- 외부 LLM 전송 시 전송범위 고지
- 운영 전환 시 내부망·보안정책 별도 검토

## 17.4 사용성

- 오류를 먼저 보여주는 기본 정렬
- 상태를 색상뿐 아니라 텍스트로 함께 표시
- 금액은 천 단위 구분
- 긴 근거는 요약과 원문을 구분
- 선택 PDF 페이지 이동 또는 페이지 이미지 보기 기능은 후속 검토

# 18. 주요 위험과 대응

## 위험 1. 기준자료 Excel 양식 다양성

**원인:** 기관별 제비율표·노임단가표의 시트명, 열 구성, 병합셀, 기준일 표기가 다름

**대응:**

- 샘플 기준으로 제비율표·노임단가표 정규화 규칙을 우선 구현
- 기준자료 인식 결과를 검증 전에 표시
- 항목·요율·단가·기준일 매핑 신뢰도가 낮으면 확인 필요 처리
- 사용자가 기준자료 유형을 직접 지정할 수 있게 함

## 위험 2. 표준품셈 PDF 표 추출 실패

**원인:** 스캔본, 복잡한 병합표, 표 선 부재

**대응:**

- 표준품셈 PDF는 선택 심화검증으로 분리
- 텍스트형 PDF를 시연자료로 우선 사용
- PyMuPDF 실패 시 pdfplumber fallback
- 근거 신뢰도가 낮으면 확인 필요 처리
- OCR은 후속 범위로 분리

## 위험 3. Excel 수식의 최근 계산값이 없음

**원인:** Excel에서 재계산하지 않고 저장했거나 다른 프로그램에서 생성

**대응:**

- 수식과 저장값 존재 여부를 별도로 표시
- 지원 규칙은 서버에서 직접 재계산
- 지원하지 않는 함수는 확인 필요 처리

## 위험 4. 항목명 표현이 제각각임

**원인:** 띄어쓰기, 약칭, 기관별 명칭 차이

**대응:**

- 별칭 사전 사용
- 유사도 기반 후보 제시
- 신뢰도가 낮으면 자동 확정하지 않음

## 위험 4. 2일 내 범위 과다

**대응:**

- 공사 핵심 요율 3~5개만 심화검증
- 용역·물품은 공통 산술검증 중심
- 시연용 표준품셈 PDF를 사전 선정
- 결과근거 화면 완성도를 기능 수보다 우선

## 위험 5. LLM의 숫자 환각

**대응:**

- LLM이 최종 금액을 계산하거나 오류 여부를 단독 결정하지 않음
- 규칙 엔진 계산결과만 판정값으로 사용
- LLM은 근거 문구 요약에만 사용

# 19. 후속 확장

## Phase 2. 공고문 자동생성

Phase 1의 `validatedProject`를 입력으로 받아 공고문 초안을 생성합니다. 이 단계에서는 원가계산서의 금액·공사명·검증상태를 다시 파싱하지 않고, 검증이 끝난 구조화 결과를 신뢰 가능한 입력으로 사용합니다.

- 공고명, 공고일, 마감일, 개찰일, 계약기간, 계약방법 입력
- `validatedProject.amounts`의 공급가액·부가가치세·도급액 자동 반영
- 검증 오류 또는 확인 필요 항목이 남아 있으면 출력 전 경고 또는 차단
- 법령·필수조항 추천
- 유사 공고문 또는 템플릿 기반 공고문 초안 생성
- 공고문 HTML 미리보기와 브라우저 PDF 출력
- AI API를 활용한 공고문 문구 추천과 법령·필수조항 설명

## Phase 3. 기준자료 관리

- 여러 제비율 Excel·노임단가 Excel·표준품셈 PDF 등록 및 버전관리
- 기준일 자동 선택
- 기준자료 추출값 수정·승인
- 공사유형·금액구간별 규칙팩

## Phase 4. 검증 범위 확대

- 표준품셈 전 항목
- 노임단가 직종 매핑
- 용역 유형별 인건비·제경비 규칙
- 물품 가격조사·복수견적 검증
- 외부 참조와 고급 Excel 함수 분석

## Phase 5. 업무시스템화

- 사용자·권한·결재
- 검증 이력 및 감사로그
- 내부망 배포
- 원본 Excel 셀 주석 또는 수정안 생성
- 원가계산서 자동 생성

# 20. 최종 제품 정의

```text
[필수 입력]
공사·용역·물품 원가계산서 Excel

[기준자료 입력]
제비율 기준 Excel
노임단가 기준 Excel

[선택 입력]
표준품셈·가격 산정지침 등 보조 기준자료 PDF

[처리]
Excel 구조·수식·값 보존 파싱
기준자료 Excel의 요율·노임단가·셀 근거 추출
선택 PDF의 페이지·원문 근거 추출
규칙 기반 재계산과 조건 비교

[핵심 출력]
웹 검증결과
+ 실제값·기대값·차액
+ Excel 시트·셀·수식
+ 기준자료 파일명·시트·셀
+ 선택 PDF 문서명·페이지·원문
+ 판단 이유와 권장조치
+ 후속 공고문 생성을 위한 validatedProject JSON
```

이 제품의 핵심은 “Excel을 텍스트로 바꾸는 것”이 아니라, **원가계산서 Excel의 계산 구조와 기준자료 Excel·선택 PDF의 근거를 연결하여 웹에서 설명 가능한 검증결과를 만드는 것**입니다.

Phase 2 공고문 자동생성은 이 검증결과를 기반으로 확장합니다. 따라서 이번 MVP의 목표는 공고문을 직접 생성하는 것이 아니라, 공고문 생성에 투입해도 되는 신뢰 가능한 원가검증 데이터와 판단근거를 만드는 것입니다.
