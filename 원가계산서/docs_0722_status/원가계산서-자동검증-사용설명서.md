# 원가계산서 자동검증 웹앱 사용설명서

- 기준일: 2026-07-22
- 대상 버전: 현재 저장소의 시연형 MVP
- 목적: 입력 파일, 자동 판단 기준, 화면에 표시되는 결과와 현재 한계를 설명한다.

> 중요: 이 문서는 현재 코드에서 실제로 동작하는 범위를 기준으로 작성했다. 제비율표 전체 조건 파싱과 세 문서 정규화 검수 화면처럼 설계만 완료된 기능은 별도로 표시한다.

## 1. 이 앱은 무엇을 하는가

원가계산서 Excel을 읽어 시트, 셀, 수식, 저장된 계산값과 셀 참조를 구조화하고 다음 내용을 자동 점검하는 웹앱이다.

- 깨진 셀 참조가 있는지
- Excel에 저장된 값과 수식을 다시 계산한 값이 일치하는지
- 공급가액, 부가가치세와 합계가 일치하는지
- 주요 보험료·제경비 항목이 수식 없이 직접 입력되었는지
- 원가계산서의 적용 요율과 제비율 기준 요율이 일치하는지
- 원가계산서의 직종별 단가와 노임단가표가 일치하는지
- 어떤 시트와 셀을 근거로 판단했는지

이 앱은 원본 Excel을 수정하지 않는다. 업로드한 파일을 브라우저에서 읽고 검증 결과만 화면에 표시한다.

## 2. 전체 처리 흐름

```text
발주 유형 선택
      ↓
원가계산서 Excel 업로드
      ↓
제비율표·노임단가표 Excel 업로드(요율·노임 검증 시 사용)
      ↓
문서 구조 인식 및 Workbook IR 생성
      ↓
인식 결과와 검증 모드 확인
      ↓
산술·수식·합계·요율·노임 검증 실행
      ↓
요약 대시보드와 항목별 판단근거 확인
```

## 3. 입력 자료

### 3.1 필수 입력

| 입력 | 형식 | 현재 용도 | 비고 |
|---|---|---|---|
| 발주 유형 | 화면 선택 | 공사·용역·물품 분류 | 기본값은 `공사`다. 현재는 공사 검증이 가장 구체적이다. |
| 원가계산서 | `.xlsx`, `.xls` | 검증 대상 | 이 파일이 없으면 자동검증 시작 버튼이 활성화되지 않는다. |

### 3.2 기준자료 입력

| 입력 | 형식 | 현재 용도 | 없어도 가능한 작업 |
|---|---|---|---|
| 제비율표 | `.xlsx`, `.xls` | 보험료·제경비 등의 기준 요율 추출 | 산술·수식·합계 검증 |
| 노임단가표 | `.xlsx`, `.xls` | 직종명별 기준 단가 추출 | 산술·수식·합계·요율 검증 |
| 표준품셈·지침 | `.pdf` | 현재는 파일 등록 정보만 보관 | 다른 모든 현재 검증 |

화면에는 제비율표와 노임단가표가 `필수`로 표시되지만, 현재 실행 로직은 원가계산서만 있어도 산술·수식·합계 검증을 허용한다. 요율이나 노임단가 비교가 필요할 때 기준자료를 함께 넣는 방식이다.

표준품셈 PDF는 현재 실제 내용을 파싱하거나 판단에 적용하지 않는다. 파일명과 크기 등의 메타데이터만 화면에 표시한다.

### 3.3 입력 파일 준비 권장사항

- 업로드 전에 Excel에서 파일을 열고 수식을 재계산한 뒤 저장한다.
- 수식 셀을 숫자로 붙여넣지 말고 가능한 한 원래 수식을 유지한다.
- `원가계산서`, `추정가격`, `공사내역`, `일위대가`, `노임단가`, `제비율`처럼 역할을 알 수 있는 시트명을 유지한다.
- 주요 항목명은 `산재보험료`, `고용보험료`, `간접노무비`, `기타경비`, `일반관리비`, `이윤` 등 일반적인 명칭을 사용한다.
- 암호가 걸린 파일, 매크로 실행이 필요한 계산, 외부 연결을 새로 갱신해야 하는 파일은 현재 지원 대상으로 보지 않는다.

## 4. 구체적인 사용 순서

### 4.1 앱 실행

프로젝트 폴더에서 다음 명령을 실행한다.

```powershell
pnpm install
pnpm run dev
```

명령이 완료되면 브라우저에서 `http://127.0.0.1:4173`을 연다.

### 4.2 파일 업로드

1. `공사`, `용역`, `물품` 중 발주 유형을 선택한다.
2. `검증할 원가계산서`에서 원가계산서 Excel을 선택한다.
3. 요율 검증이 필요하면 `제비율 기준 Excel`을 선택한다.
4. 노임단가 검증이 필요하면 `노임단가 기준 Excel`을 선택한다.
5. 표준품셈 PDF는 필요할 때 등록할 수 있지만, 현재 버전에서는 실제 검증에 사용되지 않는다.
6. `자동검증 시작`을 누른다.

파일을 읽는 데 성공하면 파일명과 함께 시트 수, 수식 수 또는 추출된 요율·직종 수가 표시된다. 읽기나 추출에 실패하면 해당 카드에 실패 메시지가 표시된다.

### 4.3 문서 인식 결과 확인

다음 화면에서 원가계산서의 각 시트가 어떤 역할로 분류되었는지 확인한다.

| 인식 역할 | 시트명 판단 예시 |
|---|---|
| 표지·요약 | 표지, 목차, 요약 |
| 원가계산서 | 원가, 계산, 추정가격 |
| 세부내역 | 내역 |
| 수량 | 수량, 물량 |
| 산출근거 | 일위대가 |
| 단가 기준 | 가격조사, 단가 |
| 노임단가 | 노임, 임금 |
| 제비율 기준 | 제비율, 요율기준 |
| 보조자료 | 위 기준에 해당하지 않는 시트 |

`구조 JSON 보기`에서는 파싱된 Workbook IR을 화면에서 확인할 수 있고, `JSON 다운로드`로 저장할 수 있다.

### 4.4 검증 모드 선택

#### 산술/합계/수식 검증만

원가계산서 자체에 저장된 값과 수식만 사용한다. 제비율표가 없거나 우선 내부 계산 오류만 확인하고 싶을 때 선택한다.

#### 산술 + 요율 검증

산술 검증에 더해 기준 요율과 원가계산서의 적용 요율·금액을 비교한다. 제비율표에서 요율이 하나 이상 추출되면 이 모드가 자동으로 선택된다.

화면의 기준요율 입력란에 값을 직접 입력하면 다음 우선순위로 적용된다.

1. 사용자가 직접 입력한 요율
2. 업로드한 제비율표에서 추출한 요율
3. 원가계산서에서 탐지한 항목의 기본값

현재 기본 요율이 내장된 항목은 부가가치세 10%이며, 다른 항목은 기준자료 또는 사용자 입력이 필요할 수 있다.

### 4.5 검증 실행 및 결과 확인

`검증 실행`을 누르면 처리 화면을 거쳐 결과 대시보드로 이동한다.

결과 화면에서는 다음 순서로 확인하는 것이 좋다.

1. 종합 결과와 상태별 건수를 확인한다.
2. `핵심 확인사항`에서 오류와 확인 필요 항목을 먼저 본다.
3. 상태, 검증 종류, 시트 필터로 결과를 좁힌다.
4. 항목을 선택해 실제값, 기대값, 차이와 근거 셀을 확인한다.
5. `상세 근거 보기`에서 계산과정과 권장 조치를 확인한다.
6. 하단 `정규화 원가계산서 리포트`에서 표준항목, 산출기초, 요율, 금액, 계산 셀과 추적 경로를 확인한다.

## 5. 앱이 Excel에서 읽는 정보

원가계산서는 단순 표가 아니라 다음 정보를 포함하는 `Workbook IR`로 변환된다.

- 파일명과 발주 유형
- 전체 시트명과 시트 역할
- 셀 주소
- 문자열·숫자·날짜·수식·오류 등 셀 유형
- 원본 수식
- Excel 파일에 마지막으로 저장된 계산값
- 화면 표시값과 숫자 서식
- 병합 범위
- 숨김 행·열 여부
- 수식에서 참조하는 같은 시트 또는 다른 시트의 셀 주소

중요하게도 이 앱은 Excel 계산 엔진을 실행하지 않는다. Excel 파일에 저장된 마지막 계산값을 읽고, 지원하는 수식만 자체적으로 다시 계산해 비교한다. 따라서 업로드 전 Excel에서 재계산하고 저장하는 것이 좋다.

## 6. 현재 자동 판단 기준

### 6.1 상태의 의미

| 상태 | 의미 | 일반적인 조치 |
|---|---|---|
| 오류 (`ERROR`) | 계산값·요율·단가가 기준과 명확히 다르거나 참조 오류가 있음 | 원본 셀과 수식을 수정해야 한다. |
| 확인 필요 (`NEEDS_REVIEW`) | 작은 차이 또는 끝수처리 가능성 등 사람이 판단해야 함 | 반올림·절사 규칙을 확인한다. |
| 주의 (`WARNING`) | 당장 값이 틀렸다고 확정할 수 없지만 구조상 위험함 | 직접 입력값이나 검사 범위를 확인한다. |
| 정상 (`OK`) | 현재 규칙과 허용 오차 안에서 일치함 | 별도 조치 없음 |
| 검증 불가 (`UNAVAILABLE`) | 기준값이나 탐지 정보가 없어 자동 비교할 수 없음 | 기준자료·요율·셀 구조를 보완한다. |

화면의 종합 결과는 오류가 하나라도 있으면 `오류`, 오류가 없고 확인 필요가 있으면 `확인 필요`, 둘 다 없으면 `정상`으로 표시된다. 주의와 검증 불가는 건수로 별도 확인해야 한다.

### 6.2 참조 오류

Excel 셀 자체가 오류 유형이면 참조 오류로 판정한다. 대표적으로 `#REF!`처럼 삭제되거나 잘못 연결된 참조가 대상이다.

- 판정: `오류`
- 근거: 오류가 발생한 시트명, 셀 주소, 표시 오류값

### 6.3 수식 자기일관성

Excel에 저장된 계산값과 앱이 같은 참조 셀을 이용해 다시 계산한 값을 비교한다.

현재 지원하는 기본 연산과 함수는 다음과 같다.

- 사칙연산: `+`, `-`, `*`, `/`
- 범위 및 집계: `SUM`, `MAX`, `MIN`
- 반올림·절사: `ROUND`, `ROUNDDOWN`, `ROUNDUP`, `TRUNC`, `INT`
- 같은 시트와 다른 시트의 직접 셀 참조

판정 기준:

| 저장값과 재계산값의 절대 차이 | 상태 |
|---:|---|
| 1원 이하 | 정상 범위 |
| 1원 초과 10원 미만 | 확인 필요 |
| 10원 이상 | 오류 |

지원하지 않는 함수, 외부 통합문서 연결, 동적 참조 또는 값을 찾을 수 없는 수식은 재계산 결과를 확정하지 않는다.

### 6.4 공급가액·부가가치세·합계

시트에서 공급가액, 부가가치세, 합계에 해당하는 라벨과 같은 행의 금액을 찾고 다음 관계를 확인한다.

```text
공급가액 + 부가가치세 = 합계
```

- 차이 1원 이하: 정상
- 차이 1원 초과: 오류

### 6.5 주요 원가 항목 탐지

현재 표준 항목 사전은 다음 항목을 중심으로 동작한다.

- 산재보험료
- 고용보험료
- 국민건강보험료
- 노인장기요양보험료
- 산업안전보건관리비
- 간접노무비
- 기타경비
- 일반관리비
- 이윤
- 환경보전비
- 안전관리비
- 부가가치세

항목명이 있는 행에서 라벨 오른쪽의 숫자 셀을 찾는다. `%` 서식 또는 0과 1 사이의 숫자를 요율 후보로 보고, 금액은 수식 셀을 우선하며 수식 셀이 없으면 가장 오른쪽 숫자 셀을 사용한다.

제비율·노임단가 시트로 분류된 시트는 원가계산서 검증 대상 항목 탐지에서 제외한다.

### 6.6 수식 누락·하드코딩

기준 요율이 필요한 보험료·제경비 항목의 금액이 수식이 아닌 숫자로 직접 입력되어 있으면 `주의`로 표시한다.

이 판정은 금액이 틀렸다는 뜻이 아니라, 산출기초나 요율이 바뀌어도 자동 반영되지 않을 위험이 있다는 뜻이다.

### 6.7 제비율 검증

요율 검증 모드에서만 수행한다.

원가계산서 금액 수식의 참조 셀 가운데 요율 셀을 제외한 금액을 산출기초 후보로 사용한다. 여러 금액 후보가 있으면 현재는 가장 큰 값을 산출기초로 선택한다.

산출기초를 찾은 경우:

```text
기대금액 = 산출기초 × 기준요율
차이 = 원가계산서 금액 - 기대금액
```

원본 수식에서 추정한 반올림·절사 규칙을 적용하며, 차이 1원 이하는 정상이고 그보다 크면 오류다.

산출기초를 찾지 못한 경우에는 원가계산서 적용 요율과 기준요율을 직접 비교한다. 기준요율이 없으면 `검증 불가`로 표시한다.

### 6.8 노임단가 검증

노임단가표에서 직종명과 그 오른쪽의 1,000 이상 숫자를 기준 단가로 추출한다. 원가계산서 전체 시트에서 공백을 제거한 직종명이 정확히 일치하는 행을 찾고, 직종명 오른쪽의 첫 1,000 이상 숫자를 적용 단가로 비교한다.

- 단가 차이 1원 이하: 정상
- 단가 차이 1원 초과: 오류
- 한 번에 최대 약 300개 적용 행을 비교

직종명이 약어이거나 서로 다른 명칭으로 작성되면 자동 연결되지 않을 수 있다.

### 6.9 검사 상한

- 기본 셀 검사 상한: 200,000개
- 정상 결과 화면 보존 상한: 200건
- 노임단가 적용 행 검사 상한: 약 300건

셀 상한을 넘으면 일부만 검사했다는 `주의` 결과가 추가된다.

## 7. 최종적으로 무엇을 출력하는가

### 7.1 문서 인식 결과

- 파일명과 업로드된 기준자료 수
- 시트별 역할과 인식 상태
- 제비율 추출 건수
- 노임단가 직종 추출 건수
- Workbook IR 화면 표시 및 JSON 다운로드

### 7.2 검증 결과 요약

- 종합 결과
- 오류·확인 필요·주의·정상·검증 불가 건수
- 우선 확인할 핵심 항목
- 상태·검증 종류·시트별 필터

### 7.3 항목별 결과

- 표준 항목명과 원본 항목명
- 원본 시트와 셀
- 원본 입력값과 적용 요율
- 시스템이 사용한 산출기초와 기준 요율
- 재계산값과 차이
- 적용한 반올림·절사 방식
- 판단 이유와 권장 조치
- 기준자료 파일명, 시트명, 셀 주소와 표시값
- 관련 셀 미리보기와 참조 셀 경로

### 7.4 정규화 원가계산서 리포트

검출된 주요 원가 항목을 다음과 같은 공통 열로 보여준다.

- 원본구간
- 비용성격
- 검증정책
- 표준항목과 원본항목
- 산출기초명과 산출기초값
- 요율과 요율출처
- 원본 금액과 계산값
- 차이
- 계산 셀
- 수식·시트 간 참조 해결상태
- 수식 안의 상수와 추적 경로
- 비고

### 7.5 현재 제공하지 않는 출력

- 검증결과 Excel 또는 CSV 다운로드
- 수정된 원가계산서 파일
- PDF 검증보고서
- 서버나 데이터베이스에 저장된 검증 이력
- 완성된 `validatedProject` 파일

현재 명시적으로 다운로드할 수 있는 파일은 문서 인식 단계의 Workbook IR JSON이다. 그 밖의 검증 결과는 웹 화면에서 확인한다.

## 8. 현재 제비율표 파싱 방식과 주의사항

현재 파서는 두 가지 방식을 순서대로 시도한다.

1. 산출기초식 헤더가 있는 매트릭스형 제비율표에서 간접노무비·기타경비·일반관리비·이윤·산업안전보건관리비 등의 첫 숫자 요율을 추출한다.
2. 매트릭스형에서 충분히 찾지 못하면 항목명과 같은 행의 첫 요율형 숫자를 추출한다.

현재 매트릭스 방식은 대표적으로 첫 번째 또는 최소 구간의 요율을 채택하고, 항목별 규칙을 한 건만 남긴다. 공사 종류, 금액, 기간, 발주 방식과 같은 전체 조건 조합을 비교하여 맞는 규칙을 선택하지 않는다.

따라서 제비율표에서 숫자가 추출되었다는 표시만으로 해당 공사에 맞는 요율이라고 확정하면 안 된다. 원본 제비율표의 공사 종류·금액 구간·기간과 화면의 셀 근거를 사람이 함께 확인해야 한다.

## 9. 현재 한계와 오판 가능성이 높은 경우

### 9.1 다른 시트의 원가 항목

직접재료비처럼 다른 상세 시트에서 계산되어 원가계산서로 들어오는 항목은 현재 표준 항목 사전과 같은 행 탐지 방식만으로 충분히 연결되지 않는다. 수식의 직접 참조 주소는 보존하지만, 여러 단계의 계산 계보를 완전히 따라가 항목 의미까지 연결하지는 못한다.

### 9.2 제비율 조건

병합된 다단 헤더, 공사 종류, 금액 구간, 기간, 조달 등급, 원·하도급 구분, 관급자재 포함 여부, 복합 계산식과 예외 조건을 완전하게 보존하지 못한다.

### 9.3 노임단가 명칭

직종명이 정확히 일치해야 하므로 약어, 괄호 설명, 띄어쓰기 외의 표현 차이가 있으면 연결되지 않을 수 있다. 숫자가 없는 미공표 직종도 현재 파서 결과에서 빠질 수 있다.

### 9.4 Excel 수식

지원 함수 밖의 수식, 이름 정의, 구조화 참조, 외부 파일 참조, `INDIRECT` 같은 동적 참조, 매크로와 외부 데이터 갱신은 완전하게 재계산하지 못한다.

### 9.5 결과 해석

`정상`은 현재 구현된 규칙과 허용 오차 안에서 일치한다는 뜻이지, 원가계산서 전체가 법적·계약적으로 완전히 적정하다는 보증이 아니다. `검증 불가`와 누락된 항목이 없는지도 함께 확인해야 한다.

## 10. 계획된 확장 기능

설계와 구현 계획은 완료되었지만 아직 실제 화면에는 반영되지 않은 기능이다.

- 검증 실행 전에 원가계산서·제비율표·노임단가표 정규화 결과를 각각 탭으로 보여주는 검수 화면
- 모든 제비율 규칙과 조건 조합 보존
- 관련 규정 시트와 규칙 연결
- 원가계산서 전체 시트에서 공사 종류·금액·기간 후보 추출
- 공사조건에 맞는 제비율 규칙 자동 선택
- 원본 오류와 미분류 셀을 포함한 파싱 커버리지 표시
- 노임단가 미공표 행 보존
- 치명 오류만 검증 실행 차단
- 향후 사용자 직접 수정 기능을 위한 별도 보정 레이어

관련 설계와 계획:

- `docs/superpowers/specs/2026-07-22-normalization-audit-design.md`
- `docs/superpowers/plans/2026-07-22-normalization-audit.md`

## 11. 간단한 사용 예시

가령 원가계산서에 다음 값이 있다고 가정한다.

```text
산출기초: 100,000,000원
원가계산서 적용요율: 3.1%
원가계산서 계산금액: 3,100,000원
제비율표에서 추출한 기준요율: 3.2%
```

요율 검증 결과는 다음처럼 해석한다.

```text
기대금액 = 100,000,000 × 3.2% = 3,200,000원
차이 = 3,100,000 - 3,200,000 = -100,000원
상태 = 오류
```

결과 화면에는 원가계산서의 금액 셀, 적용요율 셀, 제비율표에서 기준요율을 읽은 셀과 수정 권장사항이 함께 표시된다. 단, 현재 제비율 조건 매칭은 완전하지 않으므로 3.2%가 해당 공사 조건에 맞는지는 원본 표에서 추가 확인해야 한다.
