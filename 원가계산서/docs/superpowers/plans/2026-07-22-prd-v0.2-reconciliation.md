# PRD v0.2 정합 (기준자료 Excel화 + 정규화 리포트) Implementation Plan

> **For agentic workers:** 이 계획은 프론트엔드 전용 React+TS+Vite MVP를 PRD v0.2에 맞춥니다. 로컬 폴더는 정상 git repo가 아니므로 단계별 커밋은 생략하고, 각 Task 완료 후 `tsc --noEmit`로 검증합니다.

**Goal:** 이전 PRD(기준자료=PDF) 기준으로 만들어진 원가계산서 자동검증 웹앱을, 수정된 PRD v0.2(제비율·노임단가=Excel 필수, 표준품셈=PDF 선택, 판단근거=Excel 셀 중심, 정규화 원가계산서 리포트 표 신설)에 정합시킨다.

**Architecture:** 데이터 모델(`types.ts`)을 먼저 확장(evidence 겸용화, 기준자료 3슬롯, 정규화 행 타입) → 실제 검증 경로(`validation.ts` makeResult) evidence 갱신 → 정규화 행 파생 유틸 추가 → 화면 4종(Upload/Recognition/Dashboard/Detail) 수정 → 샘플 데이터(`sampleResults.ts`) Excel 근거로 교체 → 스타일 보강.

**Tech Stack:** React 19, TypeScript, Vite, SheetJS(xlsx). 순수 유틸은 `node --experimental-strip-types`로 단위 테스트 가능.

## Global Constraints

- 기준자료: 제비율 Excel(필수), 노임단가 Excel(필수), 표준품셈 PDF(선택). (PRD §0, §8.1, FR-003)
- 판단근거 1차 = 기준자료 Excel 시트·셀·표시값. 페이지는 표준품셈 PDF에만. (PRD §13.1)
- 상태 5종 유지: ERROR/NEEDS_REVIEW/WARNING/OK/UNAVAILABLE(검증 불가).
- 금액 표시는 천단위 구분, 상태는 색+텍스트 동시. (PRD §17.4)
- 기존 파일 구조/패턴 유지. 백엔드 도입 없음(프론트 전용 MVP 유지).

## File Structure

- `src/types.ts` — evidence 겸용 타입, `ReferenceFiles`, `NormalizedCostRow`, `ResolutionStatus` 추가.
- `src/utils/validation.ts` — `makeResult` evidence 빌드에 `documentType` 반영, `page` 선택화.
- `src/utils/normalize.ts` (신규) — `buildNormalizedRows(results)` 파생 유틸.
- `src/utils/results.ts` — evidence 위치 표기 헬퍼 `evidenceLocation()` 추가.
- `src/screens/UploadScreen.tsx` — 기준자료 3슬롯.
- `src/screens/RecognitionScreen.tsx` — 기준자료 인식 카드 Excel 중심 + 표준품셈 선택/미수행.
- `src/screens/DashboardScreen.tsx` — 검증불가 카드 + 핵심 확인사항 + 정규화 리포트 표 + evidence 위치 표기.
- `src/screens/DetailScreen.tsx` — 기준자료 근거 패널 Excel/PDF 분기.
- `src/data/sampleResults.ts` — evidence Excel 근거로 교체, 표준품셈/노임 샘플 추가.
- `src/App.tsx` — `pdfFiles` → `referenceFiles` 상태로 전환, props 배선.
- `src/styles.css` — 3슬롯 그리드, 핵심 확인사항, 정규화 표(가로 스크롤).
